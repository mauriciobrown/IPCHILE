Control de Versiones con GitHub

Para gestionar el control de versiones del proyecto utilizo Git y GitHub. Esto me permite trabajar de forma organizada, mantener un historial de cambios y colaborar en equipo.

  • Creación de repositorio en GitHub.
  • Commits regulares para registrar cambios.
  • Uso de ramas (branch) para organizar nuevas funcionalidades.
  • Colaboración en equipo con push y pull requests.
Flujo básico de comandos

git init
git remote add origin https://github.com/usuario/repositorio.git
git add .
git commit -m "Primer commit"
git push origin master
